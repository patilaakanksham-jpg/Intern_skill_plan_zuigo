# What an AI Coding Assistant Is (and Isn't)

## What it's good at
- Writing boilerplate code and starter templates quickly
- Explaining errors and unfamiliar code in plain language
- Giving a rough first draft of a function or script to build on
- Helping translate ideas into code structure

## Where it fails or misleads
- It can generate code that looks correct but has hidden bugs
- It doesn't actually understand my specific project unless I explain it clearly
- It won't reliably catch security issues unless I ask it to check
- Small mistakes can pile up if I accept everything without checking

## My rule
If I can't explain what a piece of code does line by line, I don't use it without checking it first. The AI helps me go faster — it doesn't replace understanding what I'm shipping.
